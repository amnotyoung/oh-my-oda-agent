# oh-my-oda-agent

> **OMO(oh-my-openagent)의 멀티 에이전트 설계 원리를 ODA(공적개발원조) 도메인에 적용하는 학습·실험 프로젝트.**
> KOICA 사업평가 업무를 보조하는 agent 팀을, [Claude Code](https://claude.com/claude-code) 위에서 작은 슬라이스부터 직접 구현해 본다.

코딩 에이전트인 OMO를 그대로 가져오는 대신, **도메인 무관 설계 원리만 이식**한다 — 역할=권한, 근거 게이트, Rules 주입, 검증, 사람 게이트.

---

## 🍰 현재: 슬라이스 1 — 효과성 평가 + 근거 검증

가장 작은 동작 단위. ODA 사업보고서를 넣으면, **효과성을 평가하고 → 그 근거를 독립 검증**한다.

```
당신(평가 요청)
   │
   ▼  [평가총괄 = 메인 Claude, CLAUDE.md가 행동 규칙 주입]
효과성 평가관 (dac-effectiveness-evaluator)   ← 읽기 전용 권한
   │  성과지표 목표 vs 실적 대조, 근거 없으면 "평가 불가"
   ▼
품질 검증자 (quality-verifier)                ← 읽기 전용 권한
   │  평가관이 인용한 근거가 원문에 진짜 있는지 대조
   ▼
사람(평가담당관)에게 초안 인계  ← 최종 등급은 사람이 확정
```

### 이 슬라이스에 담긴 OMO 원리

| OMO 원리 | 이 프로젝트의 구현 | 파일 |
|---------|------------------|------|
| **역할 = 권한** | 두 에이전트 모두 `tools: Read, Grep, Glob`로 읽기 전용 (write/edit 불가) | `.claude/agents/*.md` frontmatter |
| **근거 게이트** ("NO EVIDENCE = NOT COMPLETE") | "근거 없으면 등급 없음" 규칙을 프롬프트에 강제 | `CLAUDE.md`, 평가관 |
| **완료 주장 불신** | 독립 검증자가 근거를 원문과 대조 | `quality-verifier.md` |
| **Rules 주입** | `CLAUDE.md`가 모든 세션에 평가 원칙 자동 주입 | `CLAUDE.md` |
| **사람 게이트** (공공기관 특수) | 최종 등급 확정은 AI가 못 함, 사람 몫 | `CLAUDE.md` |

## ▶️ 써보기

1. 이 디렉토리에서 Claude Code를 연다:
   ```bash
   cd oh-my-oda-agent
   claude
   ```
2. 이렇게 요청한다:
   > `samples/sample-evaluation-report.md` 이 사업의 효과성을 평가해줘.
3. 관찰 포인트:
   - 평가관이 **취업률(68% vs 70%)**은 근거와 함께 평정하는가?
   - **직무역량(O-2)**은 보고서에 데이터가 없다 → 평가관이 **"근거 불충분 → 평가 불가"**로 처리하는가? (지어내면 실패)
   - 검증자가 평가관의 근거를 **원문과 대조**해 잡아내는가?
   - 최종적으로 **"등급은 사람이 확정"**으로 넘기는가?

> 샘플 보고서는 O-2 지표의 실적을 일부러 비워 뒀다. "근거 게이트"가 진짜 작동하는지 보는 함정이다.
