# oh-my-oda-agent

> **OMO(oh-my-openagent)의 멀티 에이전트 설계 원리를 ODA(공적개발원조) 도메인에 적용하는 학습·실험 프로젝트.**
> KOICA 사업평가 업무를 보조하는 agent 팀을, [Claude Code](https://claude.com/claude-code) 위에서 작은 슬라이스부터 직접 구현해 본다.

코딩 에이전트인 OMO를 그대로 가져오는 대신, **도메인 무관 설계 원리만 이식**한다 — 역할=권한, 근거 게이트, Rules 주입, 병렬 다각도, 검증, 사람 게이트. 평가 기준·척도는 **실제 KOICA 평가지침 2024년판** 기반(`reference/`).

---

## 🍰 현재: 슬라이스 2 — 6기준 병렬 평가팀 + 종합점수

ODA 사업보고서를 넣으면, **5개 DAC 기준 평가관이 병렬로** 각자 담당 기준을 평정하고 → 근거를 검증한 뒤 → **종합점수(20점)→A~F 등급(안)**을 산출한다.

```
당신(평가 요청)
   │
   ▼  [평가총괄 = 메인 Claude, CLAUDE.md가 행동 규칙·KOICA 기준 주입]
5개 기준 평가관 병렬  ← 읽기 전용 권한, 각자 담당 기준만 1~4점 잠정 평정
  ├ 적절성(Relevance)      ├ 일관성(Coherence)    ├ 효과성(Effectiveness)
  ├ 효율성(Efficiency)     └ 지속가능성(Sustainability)
   │   근거 없으면 "평가 불가" (지어내기 금지)
   ▼
품질 검증자 (quality-verifier)  ← 근거 원문 대조 + 점수-근거 정합성
   ▼
종합점수 산정  ← 5점 합산→20점→A~F 등급(안)
   │   ⚠️ 평가 불가 기준이 있으면 종합점수 단정 안 함 (단서부/보류)
   ▼
사람(평가담당관)에게 초안 인계  ← 최종 등급은 사람이 확정
```

*영향력(Impact)은 2024 지침대로 종료평가 종합점수에서 제외(사후평가용).*

### 이 슬라이스에 담긴 OMO 원리

| OMO 원리 | 이 프로젝트의 구현 |
|---------|------------------|
| **병렬 다각도** (hyperplan 5인 비평가) | 5개 기준 평가관이 **서로 다른 관점**으로 같은 사업을 병렬 평가 |
| **역할 = 권한** | 모든 평가관 `tools: Read, Grep, Glob` (읽기 전용), 담당 기준만 |
| **근거 게이트** | "근거 없으면 평가 불가" + 종합점수도 평가 불가 기준 있으면 단정 안 함 |
| **완료 주장 불신** | 검증자가 근거 대조 + 점수-근거 정합성 점검 |
| **Rules 주입** | `CLAUDE.md`가 KOICA 2024 기준·척도·원칙 자동 주입 |
| **사람 게이트** | 종합 등급 확정은 AI가 못 함, 사람 몫 |

관찰 포인트:
- **5개 기준 평가관이 병렬로** 떠서 각자 1~4점을 매기는가?
- 데이터가 없는 기준(예: **일관성**·**지속가능성**은 샘플에 근거 희박)을 **"평가 불가"**로 정직하게 두는가?
- 그래서 종합점수를 **억지로 A~F로 단정하지 않고** "N개 평가 가능 / M개 평가 불가"로 처리하는가?
- 검증자가 점수–근거 괴리를 잡는가? 최종적으로 **"등급은 사람이 확정"**으로 넘기는가?

- ✅ **슬라이스 2.5**: **타당성(Validity) 평가관 추가** — CTS 사업용 6번째 보조 기준(기술·재무·개발 타당성). CTS 사업은 6기준(24점)으로 평가
- ✅ **슬라이스 3**: **완료 엔진** — 평가 작업판(체크박스) + Stop hook(`.claude/hooks/boulder.sh`)으로 긴/다건 평가를 중간에 멈추지 않고 끝까지 완수. OMO Boulder 재현 (정체·상한 가드)
- ✅ **슬라이스 4 (현재)**: **평가보고서 품질심사관** — *사업*이 아니라 *보고서*를 24문항/100점/A~D로 심사하는 메타 평가 (`report-quality-inspector`, KOICA 2024 품질검토 룰브릭, 합격선 70점)
