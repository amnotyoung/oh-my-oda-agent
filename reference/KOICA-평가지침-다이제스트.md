## ⚠️ 이 지침의 성격 (반드시 먼저 읽을 것)

이 문서는 KOICA 평가의 **절차·실무 매뉴얼**이지, **평가 기준의 정의서가 아니다.** 길라잡이 스스로 원천 문서들을 "전제"·"준용"한다고 밝힌다 (p.4).

**그래서 이 지침에 ❌ 없는 것:**
- 효과성 등 각 기준의 **상세 정의**
- 사업 성과 **평정 등급척도**(매우만족~미흡 등)의 단계·정의·점수구간
- 기준별 **표준 평가질문 풀** (적절성·효율성 2개만 예시 게재, p.72)

**→ 위 내용의 원천 문서 (추가 확보 시 시스템이 완성됨):**
- 『프로젝트 종료평가 가이드라인』(2017.06)
- 『개발협력사업평가 가이드라인』(2008.11)
- 『프로젝트 기획·모니터링 및 평가방법론』(2009.11)
- 「국제개발협력 통합평가지침」·「통합평가매뉴얼」·「평가윤리지침」, 「사업평가 시행세부지침」

> **원칙:** 이 지침에 없는 등급척도·정의를 **임의로 지어내지 않는다.** 시스템의 "근거 없으면 등급 없음"을 지침 적용에도 그대로 지킨다.

## 1. 평가 기준 체계 (p.52-53, 사업평가시행세부지침 별지3호)

KOICA는 **OECD DAC 5대 기준 + 범분야 이슈**를 사용한다. (2017년 문서라 2019년 추가된 **일관성(Coherence)은 미포함**.)

| # | 평가기준 | KOICA 세부 평가항목 |
|---|---------|-------------------|
| 1 | **적절성 (Relevance)** | (1) SDGs·공여국/수원국 정책 부합 (2) 사업계획의 적절성 |
| 2 | **효율성 (Efficiency)** | (1) 자원 활용 효율성 (2) 기술적 효율성 |
| 3 | **효과성 (Effectiveness)** ★ | (1) 성과지표 측정결과 기반 사업 효과성 (2) 사업 관리의 효과성 |
| 4 | **영향력 (Impact)** | (1) 상위목표 달성 가능성 |
| 5 | **지속가능성 (Sustainability)** | (1) 자립 발전 가능성 (2) 자립 필요·제약 요소 |
| 6 | **범분야 이슈 (Cross-cutting)** | (1) 성인지적 영향성 (2) 환경 영향성 |

표준 평가질문 예시 (p.72, 출처 OECD 2007 — 적절성·효율성만 게재됨):
- **적절성**: "사업의 목표·선정·계획·실행이 협력국 및 한국의 정책·우선순위에 부합했는가? / 수혜자의 필요(needs)를 적절히 반영했는가?"
- **효율성**: "자원 투입 대비 산출(시간·예산·인력)이 효율적이었는가?"

## 2. 효과성(Effectiveness) — 시스템 핵심 (p.53)

**KOICA식 효과성 = 2축 구조:**
1. **성과지표 측정결과 기반 효과성** — 판정의 1차 근거는 **PDM 성과지표의 목표치 대비 실적치**.
2. **사업 관리의 효과성** — 산출물 전달·운영·관리 과정의 효과성.

판정 실무 원칙:
- 성과지표는 PDM 목표 대비 실적으로 측정하고 **자료출처를 반드시 명기** (p.23).
- 산출(output) 달성과 성과(outcome) 달성을 **구분**한다.
- 단순 만족도("만족하십니까? 예/아니오") 같은 약한 근거는 지양 (p.22).

> ⚠️ 효과성의 **상세 정의·등급 컷오프는 이 지침에 없음.** 원천 가이드라인 확보 전까지 등급을 단정하지 말고 잠정·정성 서술로 둔다.

## 3. 평가 수행 원칙 (AI 평가관 Rules)

| 원칙 | 내용 | 근거 |
|------|------|------|
| **근거·출처 명기** | 모든 분석에 정보출처를 명기 | p.23, p.52 |
| **미검증 = INFO 플래그** | 확인 어려운 정보는 단정 말고 `INFO`로 표기 후 다른 수단으로 검증 | p.21 |
| **강점·단점 균형** | 강점뿐 아니라 단점도 적절히 분석 | p.60 |
| **한계 명시** | 평가수행상의 한계·제약을 명확히 기술 | p.52, p.60 |
| **공정성** | 모든 주요 이해관계자 입장을 치우침 없이 포함 | p.60 |
| **독립성·중립성** | 평가 판단은 외부 영향에서 독립적, 중립적 질문 | p.21, p.25 |
| **익명 처리** | 조사대상자는 실명 대신 이니셜(담당자 A, 교사 B) | p.30 |
| **기밀유지** | 취득 정보는 사전 승인 없이 외부 공개 금지 | p.26, p.50 |
| **명료성** | 평가질문에 명확히 답하고 논리적으로 서술 | p.60 |

## 4. 평가보고서 품질심사 체크리스트 ★ (p.60) — 100점 만점

> **이 지침 최고의 자산.** 「사업평가 시행세부지침」의 평가품질관리기준(QA). 향후 **평가보고서 품질심사관 에이전트**의 룰브릭으로 그대로 이식.

| 평가항목 (배점) | 세부 체크 | 배점 |
|---|---|---|
| **1. 평가기준·원칙 답변의 명료성 (30)** | 평가질문에 명확히 답하는가 | 10 |
| | 평가원칙·기준 대비 성과를 명확히 설명하는가 | 10 |
| | 답변이 명료·논리적이어서 독자 이해를 돕는가 | 10 |
| **2. 평가조사방법의 투명성 (15)** | 평가방법론이 적절한가 | 10 |
| | 데이터·방법론이 적절한 근거를 제시하는가 | 5 |
| **3. 구성·결론의 정당성 (25)** | 구성이 논리적이고 간결한가 | 10 |
| | 결론이 객관적 논거로 뒷받침되는가 | 10 |
| | 평가수행상 한계·제약을 명확히 설명하는가 | 5 |
| **4. 내용의 공정성 (15)** | 치우침 없이 모든 이해관계자 입장 포함 | 10 |
| | 강점뿐 아니라 단점도 분석하는가 | 5 |
| **5. 평가계획 대비 충실성 (15)** | 계획상 평가질문에 논리적 결론을 제시 | 7 |
| | 계획상 방법론에 맞춰 작성됐는가 | 8 |
| **합계** | | **100** |

- **합격선: 80점.** 80점 미만 → 2차 심사 → 2차도 미만 → **미발간** (p.41, p.60).
- 심사의견 양식: 항목별 〈심사의견〉 + 〈추후 보완사항〉, 조치는 **반영/미반영/부분반영** 구분 (p.61).

## 5. 평가 절차 (5단계, p.5-6)

**① 평가기획 → ② 평가설계 → ③ 평가수행 → ④ 결과보고 → ⑤ 환류**

| 단계 | 핵심 |
|------|------|
| ① 기획 | 연간계획 의거 개별 평가계획(개요·범위·논리모형·평가매트릭스·예산·일정), RFP→입찰→계약 |
| ② 설계 | 평가매트릭스·방법·논리모형 확정, 착수보고서·착수보고회 |
| ③ 수행 | 국내조사(문헌·인터뷰·설문→자료분석) → 현지조사(KOICA는 참관자 역할) |
| ④ 보고 | 중간보고 → 평가보고서(안) → **심사(100점/80점)** → 보완 → KOICA 일괄 발간 |
| ⑤ 환류 | 제언을 이행가능(Operational) 형태로, 환류과제 지정·모니터링 |
