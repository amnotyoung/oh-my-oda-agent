# 평가 작업판 (eval-plan) — 템플릿

> 평가총괄이 긴/다건 평가를 시작할 때 이 템플릿을 `.omo/eval-plan.md`로 복사해 사용한다.
> **완료 엔진(Stop hook)**이 이 파일의 `- [ ]`(미완료)를 감지해, 하나라도 남아 있으면 평가를 멈추지 못하게 한다.
>
> - `[x]` = 완료
> - `[ ]` = 미완료 (남아 있으면 완료 엔진이 계속 일하게 만듦)
> - `[~]` = 외부 자료 대기 등으로 막힘 (사유 명기 / 미완료에서 제외됨)
>
> 여러 사업을 평가하면 사업별로 블록을 복제한다.

## 사업: <사업명> (CTS 여부: <예/아니오>)

- [ ] 자료 확인 및 사업유형 판별 (CTS면 타당성 포함 6기준)
- [ ] 적절성(Relevance) 평가
- [ ] 일관성(Coherence) 평가
- [ ] 효과성(Effectiveness) 평가
- [ ] 효율성(Efficiency) 평가
- [ ] 지속가능성(Sustainability) 평가
- [ ] 타당성(Validity) 평가  ← CTS 사업만, 일반 사업이면 시작 시 [~]로 표시
- [ ] 근거·점수 검증 (quality-verifier)
- [ ] 종합점수 산정 (5기준 20점 → A~F / CTS 6기준 24점 → 4단계 등급)
- [ ] 사람 인계용 정리 (기준별 점수·근거, 평가불가 항목, 한계)
